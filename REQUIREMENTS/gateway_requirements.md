## 2. 文档目标

明确鸿蒙网关端侧软件的开发边界、功能模块、接口规范、性能指标、验收条件与风险事项，供供应商开发、项目评审、接口联调和验收使用。

## 3. 系统定位

鸿蒙网关是公园智慧化系统的端侧控制与边缘智能节点，负责连接灯杆、星闪/蓝牙子设备、RS485 传感器、摄像机、音频设备和服务器平台。

网关向下屏蔽设备差异，向上提供标准化数据、事件和控制接口，并在断网或弱网场景下保持核心照明、AI 识别、应急联动和本地策略执行能力。

## 4. 主要集成对象

| 对象 | 关系 | 说明 |
| --- | --- | --- |
| 服务器①“闪电”智能体 | HTTP / WebSocket | 语音 ASR、复杂对话、TTS、导航触发、数字人口型/动画联动 |
| 服务器②本地 IoT 平台 | MQTT / HTTP | 设备状态、传感器数据、告警事件、照明策略、播控策略、OTA |
| 星闪 Mesh 子设备 | 星闪 / SLE Mesh | NFC 标识牌、灯控节点、信标广播和局部联动 |
| RS485 传感器 | RS485 | 光感、土壤传感器、电子水尺等 |
| 摄像机 | 本地视频流 | 边缘 AI 识别安全事件 |
| 音柱 / 屏幕 / 数字人界面 | 本地控制 / WebSocket | 广播、语音播报、导航展示、数字人动画 |

## 5. 功能需求

### 5.1 通信管家：设备接入与协议转换

目标：统一管理星闪/蓝牙子设备连接，向上提供标准接口，向下屏蔽硬件差异。

| 编号 | 需求 | 指标 / 说明 | 优先级 |
| --- | --- | --- | --- |
| GW-COM-001 | 星闪 Mesh 自动组网 | 支持全园星闪拓扑；接入 41 套 NFC 标识牌和 219 个星闪灯控节点 | P0 |
| GW-COM-002 | NFC 打卡事件接收 | 接收标识牌 ID、时间戳、手环 ID，格式化后推送服务器②；事件延迟 <100ms | P0 |
| GW-COM-003 | RS485 传感器轮询 | 光感 100ms 采样；土壤 5 分钟采样；电子水尺 1 分钟采样；各传感器独立轮询线程 | P0 |
| GW-COM-004 | MQTT 上行客户端 | 连接服务器② EMQX Broker，上报状态、传感器、告警；心跳 30 秒；断线重连；QoS=1 | P0 |
| GW-COM-005 | MQTT 下行客户端 | 订阅服务器②控制指令 Topic，解析并分发执行 | P0 |
| GW-COM-006 | 分布式软总线 | 支持相邻灯杆直连通信；优先软总线，备用 MQTT | P1 |

### 5.2 端侧 AI 推理：NPU 智能分析

目标：网关 NPU 本地运行 AI 模型，分析摄像机视频流，识别安全事件并触发联动；断网时仍可工作。

| 编号 | 场景 | 触发条件 / 输出 | 适用范围 | 优先级 |
| --- | --- | --- | --- | --- |
| GW-AI-001 | 落水识别 | 水面人体落水，置信度 >85%；输出坐标、时间、快照 | 沿湖 B 型灯杆 | P0 |
| GW-AI-002 | 烟火识别 | 烟雾/明火，置信度 >80%；联动最近灯杆广播报警 | 全型号 | P0 |
| GW-AI-003 | 摔倒识别 | 老人/儿童摔倒，置信度 >85%；生成安全告警 | A/B/C 型灯杆 | P0 |
| GW-AI-004 | 钓鱼行为识别 | 水边钓鱼，置信度 >80%；告警并音柱提示 | 沿湖 B 型灯杆 | P1 |
| GW-AI-005 | 垃圾堆积识别 | 面积 >0.5 平方米；推送环卫调度坐标 | A/B 型灯杆 | P1 |
| GW-AI-006 | 闯入禁区识别 | 非开放时段进入限制区域 | C 型入口灯杆 | P1 |

技术要求：

- 模型使用 INT8 量化。
- 单帧推理：RK3568 <50ms；RK3588 <15ms。
- 使用 RKNN Toolkit2 适配。
- 支持 AI 模型 OTA 更新。

## 6. 接口需求

### 6.1 MQTT 上行 Topic

| Topic | 用途 | 数据内容 |
| --- | --- | --- |
| `park/devices/{lamp_id}/status` | 30 秒心跳 | `online`、`firmware_ver`、`cpu_temp`、`uptime` |
| `park/devices/{lamp_id}/telemetry` | 30 分钟能耗 | `power_w`、`energy_kwh`、`brightness`、`timestamp` |
| `park/events/safety/{lamp_id}/alarm` | AI 安全事件 | `event_type`、`confidence`、`gps_lat`、`gps_lng`、`snapshot_url`、`timestamp` |
| `park/events/checkin/{lamp_id}` | NFC 打卡 | `marker_id`、`wristband_id`、`marker_type`、`timestamp` |
| `park/events/sensor/{lamp_id}` | 传感器异常 | `sensor_type`、`value`、`threshold`、`alarm_level` |

### 6.2 MQTT 下行 Topic

| Topic | 用途 | 数据内容 |
| --- | --- | --- |
| `park/commands/{lamp_id}/light` | 照明策略 | `mode`、`brightness`、`color_temp`、`schedule[]` |
| `park/commands/{lamp_id}/broadcast` | 广播任务 | `priority`、`content_type`、`content_url`、`volume` |
| `park/commands/{lamp_id}/emergency` | 应急联动 | `event_type`、`action`、`duration_sec` |
| `park/commands/{lamp_id}/ota` | OTA 更新 | `package_url`、`version`、`md5`、`module_name` |

### 6.3 HTTP 接口：网关到服务器①

| 接口 | 方法 | 用途 |
| --- | --- | --- |
| `/api/voice/asr` | POST | 上传 Opus 音频，返回识别文字与意图 |
| `/api/voice/dialogue` | POST | 上传音频，返回 TTS 流，支持 Chunked |
| `/api/navigation/trigger` | POST | 上传 `wristband_id`、`lamp_id`，返回问候语、路径和屏幕内容 |
| `/api/devices/{lamp_id}/policy` | GET | 获取照明和播控策略缓存 |

## 7. 非功能需求

| 类别 | 要求 |
| --- | --- |
| 可靠性 | MQTT 断线自动重连；核心照明、策略缓存和应急联动断网可执行 |
| 实时性 | NFC 事件 <100ms；应急插播 <200ms；本地简单语音指令 <500ms |
| 可维护性 | 网关模块应支持独立升级、日志采集、状态上报和远程诊断 |
| 可扩展性 | 新增传感器、AI 模型、Topic 和控制策略时不应破坏既有接口 |
| 安全性 | OTA 包需校验 MD5；控制指令需具备来源校验和失败回执机制 |
