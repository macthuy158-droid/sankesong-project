一、设备接入分流

先分流，再定义接口和责任边界

直接走网关

协议简单、点位清晰、复用价值高；优先沉淀为网关能力。

灯控 星闪 Mesh NFC 打卡 光感 水位 土壤 音响 屏幕 拾音器 RS485 / Modbus 点位

Assume 走网关

默认倾向网关，但必须完成协议、调试和成本评估。

简单 SOS 终端 简单水电表 摄像机状态点位 环境传感器 本地控制器 其他简单开关量

复杂系统走服务器②

厂商系统、平台型设备、视频流和复杂业务记录不下沉到网关。

停车场系统 门禁 / 闸机 海康视频 / NVR 充电桩 光伏储能 水肥一体机 水库既有设备 复杂 SOS 平台

二、总体系统链路

网关接简单设备，服务器②接复杂系统并统一对外

鸿蒙网关

  • 简单设备接入与协议转换
  • 星闪 Mesh、NFC、灯控、传感器点位
  • 本地闭环策略和边缘控制

Assume 走网关评估池

  • 协议清晰则进入网关
  • 涉及平台、历史、权限、视频则转服务器②
状态上报
控制下发

服务器② IoT 中间层

  • 复杂系统接入和厂商平台对接
  • 统一物模型、状态缓存、控制 API
  • 区 Meta 接入代理、本地共享数据入口
  • 日志、权限、联调和验收数据沉淀

复杂系统接入

  • 停车、门禁、海康、充电桩、光伏储能
  • ISAPI、GB/T28181、OCPP、DL/T645、Modbus
统一物模型 / API

服务器① 闪电智能体

  • 不直接接设备，不直接接网关
  • 通过服务器② API 获取设备状态和发起控制
  • 承载 RAG、ASR/TTS、数字人和游客互动业务

前端应用 / 运维后台 / 外部平台

  • 小程序、鸿蒙元服务、灯杆屏、瀑布屏、管理后台
  • 区 Meta 平台、大运平台、GIS/地图、指挥中心
  • 外部平台不替代 SSOT，接口以总集成规格为准

接入边界判断表

判断项 走网关 走服务器②
协议复杂度 点位清晰、Modbus/RS485/简单控制 平台 API、SDK、私有协议、视频流
数据类型 状态、告警、开关量、简单控制 记录、流水、图片、视频、权限、历史数据
复用价值 同类项目可复制，值得沉淀插件 厂商差异大，以项目交付为优先
交付风险 边缘调试可控 需要服务器日志、重试、缓存、权限和联调

关键数据流

  • 设备状态上报：设备 → 网关/服务器② → 统一物模型 → 应用/Meta
  • 设备控制下发：应用/闪电/Meta → 服务器② → 网关/厂商系统 → 设备
  • AI 业务调用：闪电智能体 → 服务器② API → 设备状态/控制
  • 运营数据沉淀：服务器②/服务器①/前端 → 数据库/日志/报表
