园区中台层 服务器② IoT 中间层 + 服务器①闪电智能体 + 共享数据层

鸿蒙网关集群

  • 端侧协议采集与控制
  • 星闪、NFC、RS485、Modbus 等接入插件
  • 本地基础闭环和断网缓存

服务器② IoT 中间层

  • 复杂系统接入、协议转换、统一物模型
  • 设备注册、权限、策略、日志、告警、API 网关
  • 向 Meta、大运平台、服务器①提供标准接口

服务器① 闪电智能体

  • AI 问答、RAG、ASR、TTS、数字人互动
  • 通过服务器② API 获取状态和发起动作
  • 不直接承担设备协议接入责任

共享数据层

  • PostgreSQL / TimescaleDB / Redis / ChromaDB
  • 设备台账、时序数据、事件、知识库、业务配置
  • 本地闭环优先，平台同步异步化
底层设备接入层 接入路径分三类；卡片颜色仅表示业务归属

直接走网关

协议清晰、点位简单、复用价值高，优先沉淀为网关插件。

鸿蒙多功能灯杆 A/B/C/D/D1/P

灯控、电源、屏幕、音频、传感和本地点位。

独立鸿蒙网关

非灯杆区域端侧接入、缓存和协议汇聚。

庭院灯 / 高杆灯 / 投光灯

灯光状态、开关、调光、策略执行。

星闪 Mesh 与运动手环

运动互动、定位联动、近场感知。

NFC / 星闪标识牌

碰一碰入口、导览、消费娱乐触点。

音响 / 屏幕 / 拾音器

灯杆互动、播报、基础媒体控制。

光感 / 水位 / 土壤等简单传感

明确寄存器、明确点位的采集设备。

RS485 / Modbus 简单设备

标准寄存器设备按插件化方式接入。

Assume 走网关

暂按网关路径设计，但必须在协议、点表、调试成本确认后落定。

简单 SOS 终端

若仅为按钮、状态、告警点位，可网关化；若含平台服务则转服务器②。

简单水电表

DL/T645、Modbus 等点表明确时可评估网关直采。

摄像机状态点位

仅状态、供电、故障等轻量点位可评估；视频流不走网关。

运动设施传感点位

跑道、健身器材等简单计数或状态信号。

气象站 / 液位 / 电子水尺

若协议简单、频率低、点表稳定，可归入网关插件。

轻量互动按钮与触发器

用于消费娱乐或导览场景的本地触发点。

复杂系统走服务器②

平台型、厂商系统、视频流、多接口联动或验收责任复杂的系统进入中台接入。

停车场系统

车牌、闸机、收费、车位、厂商平台 API。

门禁 / 闸机系统

权限、通行记录、联动策略、人员或票务系统。

视频平台 / NVR / 摄像机视频流

GB28181、ISAPI、流媒体、录像和转码。

充电桩 / 光伏 / 储能

能源设备、计费、功率策略和厂商云平台。

水肥一体机 / 水库既有设备

多设备联动、既有控制系统、验收边界复杂。

售卖机 / 商业系统

订单、支付、库存、经营数据和第三方平台。

赛事 / 活动平台接口

活动配置、成绩、排行榜和小程序业务联动。

机器人 / 无人车平台

若后续纳入，按平台 API 与安全边界接入。

接入原则

不按物理就近强制所有设备进入网关。依据协议复杂度、复用价值、开发成本、调试成本和本项目交付风险判断接入路径。

数据与闭环

服务器②负责统一物模型和平台同步；本地实时闭环优先在网关或服务器②完成，服务器①通过 API 消费能力。

风险标注

Assume 走网关设备未完成协议、点表和供应商配合确认前，不应进入固定开发报价或验收承诺。
